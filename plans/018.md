# Plan: Update Parser for ForwardingRule Changes

The parser instructions in `assets/compute/ForwardingRule.md` have been updated. We need to update `frontend/src/utils/parser.ts` to reflect these changes.

## 1. Update `GraphNode` Interface

The `GraphNode` interface in `frontend/src/utils/parser.ts` needs to be updated to include new optional fields extracted from `ForwardingRule` assets:
- `loadBalancingScheme?: string`
- `IPAddress?: string`
- `IPProtocol?: string`
- `portRange?: string`

## 2. Update `getParent` Logic

The logic for determining the parent of a `compute.googleapis.com/ForwardingRule` asset needs to be updated to support multiple parents, as per the new instructions:
- It can have a parent from `subnetwork` (priority) or `network`.
- It can *also* have a parent from `target`.
- If neither are present, it falls back to the generic parent.

We will modify the `getParent` function to return an array of parents if multiple are found.

## 3. Update `parseAssetData` Logic

We need to extract the new fields (`loadBalancingScheme`, `IPAddress`, `IPProtocol`, `portRange`) from the asset's `resource.data` and include them in the `GraphNode` object created in `parseAssetData`.
