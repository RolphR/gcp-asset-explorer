# The plan 
The following plan outlines the steps to automate the publishing of your frontend application to GitHub Pages and the GitHub Container Registry.

## 1. Nginx Configuration (`frontend/nginx.conf`)
Create an Nginx configuration file to handle Single Page Application (SPA) routing. This ensures that all requests (e.g., `/dashboard`) are directed to `index.html` so the React router can handle them.

```nginx
server {
    listen 80;
    root /usr/share/nginx/html;
    index index.html;
    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

## 2. Dockerfile (`frontend/Dockerfile`)
Create a multi-stage Dockerfile to build and serve the application:
- **Stage 1 (Builder):** Uses `node:20-alpine` to install dependencies and run `npm run build`.
- **Stage 2 (Production):** Uses `nginx:alpine` to serve the built artifacts from Stage 1 using the custom `nginx.conf`.

## 3. GitHub Actions Workflow (`.github/workflows/publish.yml`)
Create a new workflow that triggers on pushes to the `main` branch. It will contain two jobs:

**Job 1: Deploy to GitHub Pages**
- Sets up Node.js and installs dependencies.
- Builds the application with the base URL set to your repository name: `/gcp-asset-explorer/`.
- Uploads the build artifact and deploys it to GitHub Pages.

**Job 2: Publish Docker Image**
- Logs in to the GitHub Container Registry (GHCR).
- Builds the Docker image (using the default root base URL).
- Pushes the image to `ghcr.io/RolphR/gcp-asset-explorer:latest` and tags it with the commit SHA.

## 4. Prerequisites (User Action Required)
- **Enable GitHub Pages:** Go to your repository **Settings > Pages**. Under "Build and deployment", select **GitHub Actions** as the source.
