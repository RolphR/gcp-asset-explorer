# Proposed Plan

**1. Extract Parent Data (`src/utils/parser.ts`)**
- I will update the `GraphNode` interface to include a new `parent` property.
- The parser already has a `getParent()` function used internally for edge routing; I will expose this value and attach it directly to the node payload so the sidebar can read it.

**2. Next/Previous Navigation (`App.tsx` & `Sidebar.tsx`)**
- In `App.tsx`, I will maintain a flat array of all currently matched, valid assets (skipping any dimmed, hidden, or synthetic nodes).
- I will pass "Next" and "Previous" handlers to the `Sidebar` to navigate through this filtered array. 
- The sidebar header will be updated to include `ChevronLeft` and `ChevronRight` buttons, automatically disabling them when you reach the boundaries of your filtered results.

**3. Metadata Table UI (`Sidebar.tsx`)**
- I will replace the current scattered metadata tags with a clean, aligned HTML table or CSS grid at the top of the sidebar.
- It will cleanly display: **Display Name**, **Service**, **Asset Type**, **Location**, and the newly extracted **Parent**.

**4. Syntax Highlighting & Collapsible JSON (`Sidebar.tsx`)**
- To achieve high-quality syntax highlighting and collapsible object trees without reinventing the wheel, I propose installing a modern, lightweight dependency called `@uiw/react-json-view`.
- I will replace the static `<pre>` tag with this dynamic component, allowing you to easily browse massive JSON payloads by collapsing irrelevant branches.
