# Proposed Plan

**1. Create the Export Logic (`App.tsx`)**
- I will write a `handleExport` function that executes the following pipeline when triggered:
  - It filters the full graph dataset to keep only the nodes currently present in the `matchedNodeIds` set (skipping any dimmed/hidden nodes).
  - It safely filters out any "synthetic" nodes (nodes that our parser automatically generated to satisfy missing edge dependencies, which don't have a raw data payload).
  - It extracts the exact, unmodified `data` object that was originally loaded from the JSON file for each matching node.
  - It stringifies the resulting array, creates a secure, temporary Blob URL in the browser, and programmatically clicks it to trigger the download of `filtered_gcp_assets.json`.

**2. Add an Export Button (`App.tsx`)**
- In the top header next to the "Load New File" button, I will add a new "Export" button (with a `Download` icon).
- To make it clear what is being exported, the button will dynamically display the number of valid, non-synthetic assets currently matching the filters (e.g., "Export (42 Assets)").
