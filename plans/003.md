# Proposed Implementation Plan

**1. Data Model & State Management (`App.tsx`)**
- I will introduce a `FilterState` object to manage:
  - `services`: A set/array of selected GCP service groups (e.g., `compute`, `storage`).
  - `assetTypes`: A set/array of specific asset types (e.g., `compute.googleapis.com/Instance`).
  - `freeText`: A debounced text string for searching JSON properties.
  - `hideNonMatching`: A boolean toggle defaulting to `false` (Visual Dimming).
- When a new JSON file is loaded, the application will parse it and dynamically populate the filter options (the unique `services` and `assetTypes` present in the dataset).
- A `useMemo` hook will efficiently evaluate every node against the active filters. It will produce a `Set<string>` of `matchedNodeIds`.

**2. UI Component: `SearchPanel.tsx`**
- I will build a floating panel that overlays the left side of the graph canvas.
- It will contain three core filters:
  - A multiselect dropdown for **Services**.
  - A multiselect dropdown for **Asset Types**.
  - A text input for **Free Text Search**. (The search will recursively scan the raw `resource.data` of each node).
- A **Toggle Switch** at the top of the panel to control the filtering behavior: "Dim Unmatched" vs. "Hide Unmatched".

**3. Graph Visualization Updates (`GraphViewer.tsx`)**
- I will pass the computed `matchedNodeIds` and the `hideNonMatching` boolean down to the `GraphViewer`.
- **If `hideNonMatching` is `true` (Node Removal):**
  - The graph data is strictly pruned before rendering. Only nodes in `matchedNodeIds` and edges connecting two matched nodes are passed to `react-force-graph`. The graph physics engine will automatically reorganize the remaining nodes into a tighter cluster.
- **If `hideNonMatching` is `false` (Visual Dimming - Default):**
  - All nodes and edges are passed to the graph to maintain structural topology.
  - Inside the `nodeCanvasObject` custom drawing function, nodes that *are not* in `matchedNodeIds` will be rendered with an opacity of `10%`, and their text labels will be hidden.
  - Links connecting to non-matching nodes will also be rendered faintly.
