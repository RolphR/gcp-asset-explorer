# Plan: Update Parser

1. **Update `GraphNode` interface**: 
   - Change `parent?: string` to `parent?: string | string[]` to support multiple parents (required by `Address.md`).
   - Add new optional fields: `routerType` (for Router), `cidr` (for Subnetwork), `destRange` (for Route), and `status` (for Address).

2. **Update `getParent`**: 
   - Implement the specific parent extraction rules from all the `.md` files.
   - Fall back to the generic `resource.parent` or `ancestors[0]` rule.
   - Always sanitize the parent reference with `fixReference`.

3. **Add `getRelatedAssets`**: 
   - Write a recursive function to scan the entire raw asset for values that look like valid references (URLs to `googleapis.com` or paths starting with `projects/`, `folders/`, `organizations/`).
   - Sanitize with `fixReference`.

4. **Update `parseAssetData`**: 
   - Evaluate and set the new node fields (`routerType`, `cidr`, etc.).
   - Create edges to *both* the parents and all dynamically discovered related assets.

5. **Update Frontend (`App.tsx` & `Sidebar.tsx`)**: 
   - Adjust the parent filtering logic and UI to correctly render and filter nodes that have an array of parents.

## Code Changes Required

### `frontend/src/utils/parser.ts`
We will rewrite `parser.ts` to implement `getRelatedAssets`, update `getParent`, and update the `GraphNode` interface.

### `frontend/src/App.tsx`
We will update the `parentCounts` logic, `parentMatch` filter, and global options to handle `node.parent` potentially being an array.

### `frontend/src/components/Sidebar.tsx`
We will update the display of the Parent field to `Array.isArray(node.parent) ? node.parent.join(', ') : (node.parent || '-')`.
