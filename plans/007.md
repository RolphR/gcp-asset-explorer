### Proposed Plan

**1. Calculate Global Options (`App.tsx`)**
- I will create a new `useMemo` hook that runs exactly once when a new JSON file is loaded. It will parse the dataset and build exhaustive, static lists of all possible Services, Asset Types, and Locations present in the file.

**2. Initialize All Options to 0 (`App.tsx`)**
- Inside the dynamic cross-filtering logic (`filterOptions`), instead of building the counts map only from matching nodes, I will initialize the counting maps (`serviceCounts`, `assetTypeCounts`, `locationCounts`) using the exhaustive lists from the first step, explicitly setting every option's base count to `0`.
- The cross-filtering logic will then run as usual, incrementing the counts for matches.
- I will remove the logic that was specifically written to "rescue" actively selected 0-count items, as it will no longer be necessary.

**Result:** Every possible dimension value will now always remain visible in the Search Panel. Values that have been filtered out by other dimensions will simply display a `(0)` next to their name.
