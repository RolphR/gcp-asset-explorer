# Proposed Plan

**1. Sticky Header Fix (`App.tsx`)**
- I will change the main application container's layout strictly to `h-screen`. This will lock the entire application to the viewport window, ensuring that the page body never scrolls. Only the specific internal panels (like the sidebar or the filter panel) will scroll, keeping the top header permanently anchored in view.

**2. Graph Camera Focus (`GraphViewer.tsx` & `App.tsx`)**
- I will pass the ID of the currently selected node to the Graph Viewer.
- A new hook will listen for changes to this ID. When you use the "Next" or "Previous" buttons in the sidebar, it will programmatically trigger the physics engine's `centerAt()` and `zoom()` methods to smoothly animate the canvas camera directly to the newly selected node.

**3. Node Highlighting (`GraphViewer.tsx`)**
- I will update the custom canvas drawing function to check if the node currently being drawn is the selected one.
- If it is, I will draw a bright, glowing golden halo ring around the node's circle to make it stand out instantly from the rest of the graph.

**4. Resizable Sidebar (`Sidebar.tsx`)**
- Standard CSS `resize` doesn't work well for panels anchored to the right side of the screen (pulling the corner expands it off-screen). 
- Instead, I will implement a custom, invisible drag-handle along the left edge of the sidebar. You will be able to click and drag the left edge to smoothly expand or shrink the sidebar horizontally to your desired width.

**5. Prevent JSON Abbreviation (`Sidebar.tsx`)**
- I will add a `shortenTextAfterLength={100000}` prop to the `@uiw/react-json-view` component. This disables the default truncation behavior, ensuring that long raw string values (like massive IAM policy bindings or metadata keys) are always rendered in full.
