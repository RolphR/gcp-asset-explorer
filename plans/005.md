# Proposed Plan

**1. Location Dimension Integration**
- I will update the `FilterState` to include `locations: Set<string>`.
- I will add a new "Locations" filter section in the `SearchPanel` (using a map-pin icon) that functions identically to the Services and Asset Types filters.

**2. Dynamic Counters & Hiding Empty Options**
- Currently, the options are static lists of strings. I will upgrade them to an array of objects: `{ value: string; count: number }`.
- In `App.tsx`, I will implement a single-pass cross-filtering algorithm using `useMemo`:
  - To calculate the counts for *Services*, the app will check every node against the active *Location*, *Asset Type*, and *Free Text* filters. If the node passes those, it increments the counter for its Service. (The same applies inversely for calculating Location counts and Asset Type counts).
  - If a dimension value ends up with a count of `0`, it will be completely hidden from the list.
  - *Edge case handled:* If a dimension is currently checked by the user but its count drops to `0` due to another filter being applied, I will keep it visible in the list so the user can uncheck it.

**3. Resizable Filter Panel**
- I will add the CSS `resize-x` and `overflow-hidden` properties to the `SearchPanel` container.
- I will give it a `min-w-[20rem]` and a max-width constraint (e.g., `max-w-[50vw]`) so you can grab the right edge of the panel and drag it to expand or shrink the width.

**4. UI Enhancements**
- In the `SearchPanel`, I will update the rendering loops to display the new counters immediately after each checkbox label in the format: `ComputeEngine (42)`.
- The "Clear All Filters" button will be updated to also wipe the active Location selections.
