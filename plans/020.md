# The plan

Based on my analysis of the codebase, I have formulated a plan to implement the requested **List View** feature.

## Plan Overview

The goal is to add a table-based view for assets alongside the existing graph view. This involves creating a new `ListView` component, a column configuration utility, and updating the main application to toggle between views.

## 1. New Utility: Column Management
I will create `frontend/src/utils/columnUtils.ts` to handle data extraction:
*   **`flattenObject(obj)`**: Recursively flattens nested JSON objects (e.g., `{ resource: { data: { network: "..." } } }` becomes `resource.data.network`).
*   **`getAllColumns(nodes)`**: Scans all available nodes to dynamically discover every unique field across the dataset, enabling the "All exposed fields" requirement.
*   **`getCellValue(node, path)`**: Safely retrieves values for table cells, handling nested properties and arrays.

## 2. New Component: List View (`ListView.tsx`)
A new component `frontend/src/components/ListView.tsx` will feature:
*   **Table Interface**: Displays assets in rows with sortable headers.
*   **Sorting**: Supports ascending/descending sort on any column.
*   **Custom Columns**: Defaults to core fields (`displayName`, `assetType`, `location`, `group`) but allows adding any discovered field.
*   **Toolbar**: A "Configure Columns" button to open the management modal.

## 3. New Component: Column Configuration (`ColumnConfigModal.tsx`)
A modal component `frontend/src/components/ColumnConfigModal.tsx` to:
*   **Search**: Filter the list of all available fields.
*   **Toggle**: Checkboxes to show/hide columns.
*   **Reorder**: Buttons (Move Up/Down) to change column order in the table.

## 4. Integration (`App.tsx`)
I will modify `frontend/src/App.tsx` to:
*   **State**: Add `viewMode` (`'graph'` or `'list'`).
*   **UI**: Add a toggle button (Icons: List vs. Network) in the header.
*   **Logic**: Pass the *filtered* `visibleData` to `ListView`, ensuring the existing Search Panel filters apply to the list view as well.

## Verification
*   Verify that switching views preserves the current filter state.
*   Confirm that deep fields (e.g., `resource.data.network`) can be added and sorted correctly.
*   Ensure performance remains acceptable when scanning fields for column configuration.
