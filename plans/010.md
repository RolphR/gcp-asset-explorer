# Proposed Plan

**1. Add "Parent" Dimension Filter**
- I will update `App.tsx` and `SearchPanel.tsx` to include `parents` in the `FilterState`.
- `App.tsx` will extract and cross-filter the `parent` property exactly like it does for Location, Service, and Asset Type.
- I will add a new `DimensionFilter` block for Parents (using a hierarchy/network icon) to the Search Panel.

**2. Automatic Graph Rebalancing & Zoom-to-Fit**
- I will attach a reference (`useRef`) to the underlying WebGL canvas component (`ForceGraph2D`) in `GraphViewer.tsx`.
- I will add a `useEffect` hook that listens for changes to the active graph data.
- When the "Hide Unmatched" filter mode is active and the data changes (e.g., when you check a new filter box), the hook will programmatically call `d3ReheatSimulation()` to wake up the physics engine and spread the remaining nodes evenly, followed instantly by `zoomToFit(400)` to smoothly animate the camera to frame all visible nodes.

**3. Collapsible Dimensions & Vertical Resizing**
- I will update the `DimensionFilter` component to include an `isCollapsed` state. Clicking the dimension header (which will now feature a chevron arrow) will collapse the local search bar and checkbox list, allowing you to save vertical space.
- I will change the CSS of the main `SearchPanel` container from `resize-x` to `resize` (both horizontal and vertical), allowing you to pull the bottom-right corner downwards to see more checkboxes at once.
