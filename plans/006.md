# Proposed Plan

**1. Create a Reusable Dimension Filter Component**
To handle all these new features without making the code messy, I will extract the dimension checklist logic into a new internal React component (`DimensionFilter`) inside `SearchPanel.tsx`. It will manage its own local state for text filtering and sorting.

**2. Local Dimension Text Search**
- I will add a small text input directly below each dimension header (Locations, Services, Asset Types).
- As you type in this field, it will filter the visible checkboxes using a case-insensitive substring match. 
- *Crucially*, this local text field will **only** hide checkboxes from the list to help you find a specific value; it will *not* alter the global graph filtering state.

**3. Advanced Sorting Options**
- I will add a small sort button (with an icon) next to the dimension header. Clicking it will cycle through four sort modes:
  1. **Count Descending (Default):** Largest counts first. Ties are broken alphabetically by name (ascending).
  2. **Count Ascending:** Smallest counts first. Ties broken alphabetically.
  3. **Name Ascending:** Pure alphabetical (A-Z).
  4. **Name Descending:** Reverse alphabetical (Z-A).

**4. Context-Aware "Toggle All" Button**
- I will add a "Check All / Uncheck All" button (using a list-check icon) next to each dimension header.
- The logic will dynamically evaluate the *currently visible* checkboxes (those that match your local dimension text search).
  - If **any** of the visible options are currently deselected, clicking the button will **Select All** of them.
  - If **all** of the visible options are already selected, clicking the button will **Deselect All** of them.
