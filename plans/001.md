# Proposed Plan: GCP Asset Explorer Frontend

**1. Architecture & Tech Stack**
- **Framework:** React with Vite and TypeScript. This provides a fast, modern, and fully static frontend application without requiring a backend server.
- **Styling:** Tailwind CSS for rapid UI development.
- **Visualization Library:** `react-force-graph-2d` (or `3d`). This is a WebGL-backed force-directed graph library that excels at rendering large, interconnected datasets typical of GCP Asset Inventories (often 1000+ nodes).
- **Icons:** `lucide-react` for clean, modern UI iconography.

**2. Data Processing Strategy (The "TypeScript Port")**
We will implement a robust TypeScript parser that replicates the core logic found in the Python reference (`reference_implementation/assets/`).
- **Base Logic:** We'll implement a central parser that handles the generic fields (like `name`, `assetType`, `location`, `resource.data`) and the default edge resolution (parent/ancestor references) defined in `_base.py`.
- **Custom Edge Overrides:** Instead of 36 separate files, we will create a consolidated `AssetParserRegistry` in TypeScript. This will use a `switch` statement (or map) keyed by the `assetType` to handle the specific edge overrides (like extracting `network` dependencies for `compute.py` or subnets for `ComputeSubnetwork`). This keeps the frontend codebase clean while maintaining parity with the Python reference.

**3. UI/UX Flow & Key Features**
- **Upload Screen:** The app will initialize with a clean, full-screen dropzone allowing the user to upload or drag-and-drop their `inventory.json` file.
- **Interactive Graph Canvas:** Once parsed, the JSON will be rendered on a full-screen, interactive WebGL canvas.
  - Nodes will be color-coded by GCP service (e.g., Compute, Storage, IAM).
  - The canvas will support zooming, panning, and dragging nodes.
  - Directional arrows will clearly show the dependencies (edges) between resources.
- **Details Sidebar:** Clicking on any node will open a right-hand sidebar. This panel will display:
  - High-level metadata (Display Name, Asset Type, Location).
  - A formatted, scrollable JSON viewer containing the raw `resource_data` for deeper inspection.
- **Controls Panel:** Basic controls overlaying the canvas (e.g., "Recenter", "Search Node by Name", "Filter by Asset Type").

**4. Implementation Steps**
1. Scaffold the Vite React TS project and install dependencies (`react-force-graph`, `tailwindcss`).
2. Build the `AssetParser` utility to convert the raw JSON into the `nodes` and `links` structure required by the graph library, ensuring the custom GCP service logic is accurately ported.
3. Build the core UI layout components (Dropzone, Sidebar, Top Navigation).
4. Integrate the `react-force-graph` component and wire it up to the parsed data state and the sidebar selection state.
5. Add final polish (color palettes for nodes, graph physics tuning, loading states).
